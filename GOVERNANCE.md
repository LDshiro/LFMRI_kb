# Governance

この文書は、低磁場MRIナレッジベースの運用規約です。

## 1. 正本

本リポジトリを正本とします。ChatGPT Library、ChatGPT Project、ローカルメモ、Zotero、外部ストレージは補助的な作業場所です。

## 2. ChatGPT生成物の扱い

ChatGPTが生成した要約、表、草稿、仮説、コードは、初期状態ではすべて `draft` とします。以下のいずれかに該当する記述は、原典確認が終わるまで `reviewed` 以上へ昇格できません。

- 診断性能、感度、特異度、AUC、非劣性などの臨床的主張
- SNR、分解能、撮像時間、磁場強度、勾配性能、SARなどの数値
- 医療機器規制、保険、臨床導入、安全性、禁忌に関する記述
- 特定製品、企業、装置、データセットの最新状況
- 既存研究の優先権や新規性に関する主張

## 3. レビュー責任

| 項目 | 最低限の確認 |
|---|---|
| 文献ノート | DOI / PMID / URL、対象、方法、主結果、限界 |
| KBページ | 複数文献との整合性、出典リンク、用語の一貫性 |
| エビデンステーブル | 数値、単位、条件、抽出箇所 |
| 実験ノート | 実験条件、再現手順、依存環境、出力先 |
| プロンプト | 出力形式、確認手順、危険な断定の抑制 |

## 4. ブランチ運用

```text
main                         安定版。共有可能な状態。
develop                      次回リリース候補。
paper/<year-author-keyword>  個別論文ノートの追加。
topic/<area>                 分野別のKB更新。
experiment/<name>            実験・解析・シミュレーション。
maintenance/<task>           形式修正、索引生成、CI調整。
```

## 5. Pull Request要件

Pull Requestでは、`.github/pull_request_template.md` のチェックリストを使用します。少なくとも以下を満たす必要があります。

- 主要な主張に出典がある。
- 文献情報にDOI、PMID、URLのいずれかがある。
- ChatGPT生成物が未確認のまま `reviewed` や `stable` になっていない。
- 新規ファイルにfrontmatterが付いている。
- 関連するKBページ、文献ノート、エビデンステーブルが相互リンクされている。

## 6. Codexのテスト実行とgit更新

Codexは、作業内容に応じて以下の非破壊的な確認を自律的に実行してよいものとします。

- `make check`、`make index`
- frontmatter確認、BibTeX確認、索引生成
- 差分確認、状態確認、テストや検証コマンドの実行

ただし、gitの状態を更新する操作は、Codexが結果を要約してユーザーへ共有し、明示的な承認を得た後に限ります。ここでいう git更新 には、少なくとも以下を含みます。

- `git add`
- `git commit`
- `git push`
- `git tag`
- ブランチ作成やブランチ切替で、作業状態に影響するもの

運用ルールは以下とします。

- Codexは、実行したテストや検証の結果を要点だけ共有する。
- ユーザーが承認した場合のみ、Codexは必要な git 更新を行う。
- テスト失敗時は、失敗内容と影響範囲を共有し、修正または承認なしに git 更新しない。
- 破壊的な操作は、別途明示的な指示がない限り行わない。

## 7. リリース

月次または四半期ごとにタグを作成します。

```text
vYYYY.MM        月次スナップショット
vYYYY.QN        四半期レビュー版
```

リリース時には、以下を更新します。

- `CHANGELOG.md`
- `logs/review-log.csv`
- `kb/index.md`
- `literature/reading-list.md`
- `docs/roadmap.md`

## 8. 非公開情報

以下は公開リポジトリに含めないでください。

- 患者情報、匿名化が不十分なDICOM、臨床画像
- 契約・NDA下の装置仕様、未公開共同研究資料
- 著作権上再配布できないPDF
- ChatGPTのraw export zip / json / html
- 個人的なメール、会議録、認証情報、APIキー

## 9. 廃止と訂正

誤りを見つけた場合は、該当ページを直接削除せず、可能な限り以下を記録します。

- 誤りの内容
- 発見日
- 根拠
- 修正内容
- 置換先ページまたはPR

`deprecated` にする場合は、frontmatterの `replaced_by` または本文中の置換先を明記します。
